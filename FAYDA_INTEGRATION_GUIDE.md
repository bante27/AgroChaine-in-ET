# Fayda ID (Ethiopian National ID) Integration Guide

This guide outlines the steps required to transition from the current manual verification system to a real-time National ID verification using the **Fayda ID** system.

## 1. Choose an Integration Method

There are two primary ways to verify Fayda IDs:

### Option A: Smile ID (Recommended)
Smile ID is an official KYC provider that has direct integration with Ethiopia's National ID Program (NIDP).
*   **Pros**: Documentation is in English, easy-to-use SDKs, includes Face Match (comparing user's selfie to ID photo).
*   **Website**: [https://usesmileid.com/](https://usesmileid.com/)
*   **Service**: "Ethiopia ID Verification"

### Option B: Direct NIDP Integration
Integrate directly with the Ethiopian National ID Program APIs.
*   **Pros**: No middleman fees.
*   **Website**: [https://id.et/](https://id.et/)
*   **Developer Portal**: [https://developer.id.et/](https://developer.id.et/)

## 2. Steps to Apply for Credentials

### For Smile ID:
1.  **Sign Up**: Create an account on the Smile ID portal.
2.  **Compliance Review**: Submit your business license and use case description (AgroChain Ethiopia).
3.  **API Keys**: Once approved, go to the "API Keys" section in your dashboard.
4.  **Environment**: 
    *   `SANDBOX`: For testing.
    *   `PRODUCTION`: For real users.

### For Direct NIDP (Fayda):
1.  **Legal Registration**: Ensure your business is legally registered in Ethiopia.
2.  **Request Access**: Contact NIDP via `info@id.et` or visit their office at the Ministry of Innovation and Technology (MinT).
3.  **Submit Use Case**: Explain how you will use the data and ensure user privacy.
4.  **Integration Approval**: They will provide a Client ID and Client Secret for their OAuth2 server.

## 3. Implementation Logic

Once you have your `API_KEY` or `CLIENT_SECRET`, update the backend logic in `server/routes/userRoutes.js`.

### Example Logic (Pseudocode):
```javascript
const verifyNationalID = async (idNumber) => {
  const response = await axios.post('https://api.fayda.gov.et/verify', {
    id: idNumber,
    api_key: process.env.FAYDA_API_KEY
  });
  
  return response.data; // { status: "Verified", fullName: "..." }
};
```

## 4. Security & Compliance
*   **Encryption**: Never store National ID numbers in plain text if possible. Consider hashing or encrypting them at rest.
*   **Privacy**: Inform users that their data is being checked against the National ID database for security purposes.
*   **Data Minimization**: Only request the fields you absolutely need (Full Name, Date of Birth).
